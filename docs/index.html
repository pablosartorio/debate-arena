Un cuarto pixel-art donde el debate sucede, token a token.

La aceleración tecnológica es nuestra mejor apuesta: frenar la IA hoy es condenar el futuro.
Cada gran tecnología prometió liberar y terminó concentrando poder. ¿Quién decide?

▸ burbujas, estados y scoreboard flotan sobre la escena, alineados al cast pintado.

QUÉ HACE

Un pipeline agentic completo, no solo dos chatbots tirándose texto.

⚔️

DEBATE MULTI-AGENTE

Dos personas con ideologías opuestas argumentan en turnos alternados, respondiendo a los puntos del otro.

⚖️

MODERADOR CON SCORING

Evalúa cada turno en 8 dimensiones (fidelidad factual, alucinación, claridad…) y arma un scoreboard en vivo.

🔭

SCOUT + BÚSQUEDA WEB

Antes de arrancar, analiza el tema y trae contexto real desde DuckDuckGo para calibrar el debate.

🧠

PLANIFICACIÓN INTERNA

Cada agente arma su estrategia y sus afirmaciones en una fase privada (JSON) antes de hablar.

📡

STREAMING EN VIVO

Tokens, evaluaciones e intervenciones viajan por WebSocket en tiempo real, palabra por palabra.

💾

HISTORIAL + EXPORT

Todo se persiste en SQLite: revisás debates pasados, sus scores e intervenciones, y los exportás en JSON.

CÓMO FUNCIONA

Un grafo LangGraph recorre el debate de punta a punta.

SCOUT ▸ PLAN▸ SPEAK▸ MODERATE▸ ¿INTERVENE? × cada turno ▸ SUMMARY

Elegís tema y participantes. El Scout mapea el terreno: conceptos clave, riesgos de desinformación y criterios de evaluación.

Por turno, cada agente planifica su jugada, habla en streaming, y el moderador lo evalúa y actualiza el score.

Si un turno se va de eje (alucinación, repetición, evasión), el moderador interviene como tercer hablante para reencauzar.

Al cerrar: ganador por score acumulado (determinista) + un resumen narrativo del LLM. Todo queda guardado.

PROBALO LOCALMENTE

Todo corre local con Docker + Ollama. Sin API keys.

# 1. clonar
git clone https://github.com/pablosartorio/debate-arena.git
cd debate-arena

# 2. levantar (Ollama + backend)
make dev

# 3. descargar el modelo (1ra vez) y abrir
./init_models.sh
open http://localhost:8080
