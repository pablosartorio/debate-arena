◆ DEBATE ARENA ◆

Dos IA debaten en tiempo real. Un tercer agente modera, evalúa e interviene.

Sistema de debate automático entre agentes de IA con interfaz pixel-art. Dos agentes con posturas ideológicas distintas debaten un tema en vivo, orquestados por un grafo LangGraph y moderados por un tercer agente evaluador.

PYTHON 3.12 FASTAPI LANGGRAPH 0.2 OLLAMA AIOSQLITE · WAL VANILLA JS DOCKER

¿Cómo funciona, en 30 segundos?

  1. El browser abre un WebSocket y manda start con el tema.
  2. ws_bridge.py lanza el grafo LangGraph y drena sus eventos a una asyncio.Queue.
  3. El grafo recorre scout → router → plan → speak → moderate → [intervene] → summarize → finalize.
  4. Cada token, score e intervención viaja al frontend en vivo; todo se persiste en SQLite.

Arquitectura

Tres capas: el browser (WebSocket + REST), el bridge (ws_bridge.py) que media entre el WebSocket y el grafo, y el grafo LangGraph que ejecuta el debate.

El bridge WebSocket ↔ LangGraph

DebateSession arranca dos tasks concurrentes: _run_graph() que ejecuta graph.ainvoke(), y _drain() que consume la asyncio.Queue y reenvía cada evento con ws.send_json(). Los nodos nunca tocan el WebSocket: solo hacen queue.put(...).

sequenceDiagram
  participant B as Browser
  participant WS as FastAPI /ws
  participant S as DebateSession
  participant Q as asyncio.Queue
  participant G as LangGraph
  B->>WS: start {topic, model, ...}
  WS->>S: session.start(payload)
  S->>G: create_task(_run_graph → ainvoke)
  S->>Q: create_task(_drain)
  loop por cada evento
    G->>Q: put(event)
    Q->>B: ws.send_json(event)
  end
  G->>Q: put(SENTINEL)
  B->>WS: stop
  WS->>S: cancel()
  S-->>G: task.cancel() + status=stopped
POR QUÉ ESTE PATRÓN
Desacopla el grafo del transporte: si el WebSocket se cae, el grafo no crashea. Hay backpressure (queue acotada), un sentinel {"__sentinel__":"end"} que cierra el drain, y un timeout de drenado.

El grafo del debate

8 nodos conectados por aristas, algunas condicionales (graph/edges.py). Este diagrama refleja el cableado real de graph/graph.py:

flowchart LR
  START([START]) --> scout
  scout --> router
  router -->|speak| plan
  router -->|end| summarize
  plan --> speak
  speak -->|moderate| moderate
  speak -->|end| summarize
  moderate -->|intervene| intervene
  moderate -->|router| router
  moderate -->|end| summarize
  intervene --> router
  summarize --> finalize
  finalize --> DONE([END])
FIX DE FAIRNESS
El cierre por "turnos agotados" lo decide route_after_moderate, después de moderar. Así el último turno también se evalúa y los cumulative_scores quedan parejos entre ambos agentes.

Responsabilidad de cada nodo

NodoQué hace
scoutanaliza el tema 1 sola vez; genera contexto + criterios; corre web_search si hay tools
routerdecide a quién le toca (alternancia current_turn % len(turn_order))
planel agente planifica su estrategia (JSON privado, no se emite)
speakel agente genera su discurso (streaming de tokens)
moderateevalúa el turno en 8 dimensiones y decide si intervenir
interveneel moderador habla cuando detecta un problema
summarizedetermina el ganador por score y pide la narrativa al LLM
finalizecierra el debate y emite conversation_end

Degradación elegante

Si un agente (scout / plan / moderate / summary) falla o supera su timeout, el grafo continúa con un valor por defecto (plan vacío, score neutro, summary mínimo). El debate nunca se traba por una llamada LLM caída.

CONTRATO ADITIVO
Los eventos solo agregan campos; nunca renombran ni eliminan. El frontend ignora lo que no conoce.

Agentes & Personas

Los debatientes hablan vía BaseAgent (streaming contra /api/chat de Ollama, con cap de palabras). Los agentes especializados (ScoutAgent, ModeratorAgent, SummaryAgent y plan_for_turn) usan salida JSON validada con Pydantic.

MODELO EFECTIVO
El modelo elegido en la UI (state["model"]) se usa en todos los agentes, incluido el discurso de los debatientes (fallback: el modelo de la persona).

Scoring

Cada turno se evalúa en 8 dimensiones (0.0–1.0). El total es un promedio ponderado (SCORE_WEIGHTS), invirtiendo las dimensiones de penalización.

DimensiónPesoNota
factual_fidelity0.20
hallucination_risk0.20invertida (1 − hr)
consign_compliance0.15
rebuttal_quality0.15
clarity0.12
role_adherence0.10
repetition_penalty0.05invertida (1 − rp)
tool_usage_quality0.03

Ganador

Se determina deterministicamente sumando los total por agente (determine_winner), con un margen (0.05) para declarar empate técnico. El LLM del summary_node solo aporta la narrativa, no elige ganador.

Intervención

Además del flag del LLM, hay thresholds deterministas en config.MODERATION_THRESHOLDS que disparan una intervención aunque el LLM no la pida:

hallucination_risk  > 0.75  → correction
repetition_penalty  > 0.80  → warning
consign_compliance < 0.30  → redirect
role_adherence     < 0.40  → warning

Tools

Si enable_tools=true, el scout puede usar tools externas. Cada tool hereda de BaseTool, implementa run() async y nunca lanza: ante un fallo devuelve ToolOutput(success=False, ...).

Tool incluida

web_search

Búsqueda en DuckDuckGo (sin API key), corrida en threadpool para no bloquear el event loop. Devuelve los top snippets + URLs como fuentes.

Cómo agregar una tool

from tools.base import BaseTool, ToolInput, ToolOutput

class MiTool(BaseTool):
    name = "mi_tool"
    description = "Qué hace"
    requires_network = False

    async def run(self, input: ToolInput) -> ToolOutput:
        return ToolOutput(success=True, result="...", source="mi_tool")

Después registrarla en tools/registry.py y, si querés que el scout la sugiera, referenciarla desde el scouting.

Decisiones de arquitectura (ADR)

ADR 001 — LangGraph para orquestación

El debate tiene fases secuenciales con lógica condicional y puede cortarse en cualquier momento. LangGraph (StateGraph) hace las bifurcaciones declarativas y el estado fluye solo entre nodos. Costo: una dependencia externa y una curva de aprendizaje. Leer ADR →

ADR 002 — asyncio.Queue como bridge WS ↔ grafo

Una asyncio.Queue desacopla el grafo del WebSocket: backpressure, cancelación limpia y nodos testeables sin socket. El sentinel {"__sentinel__":"end"} marca el fin del stream. Leer ADR →

Roadmap

El proyecto es un sandbox de aprendizaje de LangGraph; el detalle está en notas/planes.md. Líneas de exploración:

B1 · Memoria cross-debate

El scout lee debates anteriores del mismo tema (memory store de LangGraph).

B2 · RAG

Vectorizar documentos y recuperarlos por similitud antes del debate.

B3 · Humano en el loop

Un nodo que pausa y espera input del usuario (interrupt()).

B5 · Streaming nativo

Reemplazar el bridge ad-hoc por .astream_events() de LangGraph.

C · Observabilidad

LangSmith o tracing propio sobre la tabla events.

D · Modo torneo

Varios debates con distintas combinaciones; orquestador de orquestadores.
